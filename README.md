# FocusFlow_Lite
这是用于FocusFlow_Lite的项目仓库

仓库内分别有三个文件夹
- Arduino UNO Q用于存放烧写到对应开发板内的代码
- Laptop 用于存放在笔记本端运行的程序
- ESP32-C3 用于存放烧写到ESP32手环上的程序

---
以下重点说明整个系统的工作方式；

1. UNO Q向ESP32-C3手环发送数据
   使用GATT服务单向发送数据，参考文档已经上传，后续直接根据参考文档在UNO Q上运行蓝牙服务端即可；数据包格式只发送强度和振动次数信息。
2. UNO Q与笔记本的双向通信
   笔记本端需要通过蓝牙向UNO Q以一定频率发送MediaPipe跑出来的头部位姿信息（一个特征向量）以及屏幕识别结果和走神事件判断。
   同时笔记本要和UNO Q同步状态，表明程序运行状态（学习or工作监控状态/休息/未运行）
   有一个难点是需要让AI设计一份完整的 打通开发板和笔记本端的通信协议。
3. 走神判断指标
   如果Mediapipe端识别到用户的视线严重偏离屏幕并且持续一段时间（例如3s），笔记本直接向UNO Q发送走神事件标识；
   如果屏幕识别到用户正在摸鱼，笔记本直接向UNO Q发送摸鱼事件标识；
   其他状态下使用特征向量和实时波形进行较为鲁棒性的判断。我们缺乏真实数据，可能无法给出一个较好的分类器。
4. tft彩屏GUI设计
   要求显示信息已经在之前的文档中有所说明。我个人建议当走神和摸鱼事件触发时，界面直接不显示当前的专注力分数，而是直接显示提示画面。
   在正常状态下显示专注程度
